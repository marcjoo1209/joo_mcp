# 01. MCP란 무엇인가?

> 대상: MCP를 한 번도 들어본 적 없는 사람

## 한 줄 정의

**MCP(Model Context Protocol)** 는
AI 모델(예: Gemini, Claude)이 **외부 도구·데이터에 연결**할 수 있게 해주는
**표준 규약(protocol)** 입니다.

쉽게 말해 **"AI를 위한 USB-C 포트"** 입니다.
- USB-C는 충전기·모니터·외장하드를 노트북에 같은 방식으로 꽂게 해줍니다.
- MCP는 DB·파일·API·각종 도구를 AI에 같은 방식으로 꽂게 해줍니다.

---

## 왜 필요한가? (MCP가 없던 시절의 문제)

AI 모델은 똑똑하지만, 기본적으로는 **자기가 학습한 지식 안에 갇혀 있습니다.**
모델 혼자서는 아래를 못 합니다.

- 우리 회사 DB에서 오늘 주문 건수 조회하기
- 내 메모를 실제로 저장/삭제하기
- 외부 날씨 API 호출하기

그래서 개발자는 "AI가 도구를 쓰게" 만들어야 하는데,
예전에는 **AI마다, 도구마다 연결 코드를 따로따로** 작성해야 했습니다.

```
        Gemini ── (전용 코드) ── DB
        Gemini ── (전용 코드) ── 파일시스템
        Claude ── (또 다른 전용 코드) ── DB
        Claude ── (또 다른 전용 코드) ── 파일시스템
```

도구 M개 × 모델 N개 = **M×N개의 연결 코드.** 끔찍하죠.

## MCP가 해결하는 방식

MCP는 그 사이에 **공통 규약**을 둡니다.
도구는 "MCP 서버"로 한 번만 만들면, MCP를 지원하는 **어떤 AI든** 꽂아 쓸 수 있습니다.

```
                 ┌──────────────┐
        Gemini ──┤              ├── DB (MCP 서버)
                 │   MCP 규약    │
        Claude ──┤              ├── 파일시스템 (MCP 서버)
                 └──────────────┘
```

M + N개만 만들면 됩니다. (M×N → M+N)

---

## MCP 서버가 AI에게 제공하는 3가지

| 종류 | 설명 | 이 예제에서 | 비유 |
|---|---|---|---|
| **Tools (도구)** | AI가 호출해 **행동**하는 함수 | `create_note`, `delete_note` 등 | 동사 (만들어라, 지워라) |
| **Resources (자원)** | AI가 읽는 데이터 | (이 예제는 도구로 통일) | 명사 (파일, 레코드) |
| **Prompts (프롬프트)** | 미리 정의된 작업 템플릿 | (사용 안 함) | 양식, 매크로 |

이 예제에서는 가장 중요하고 직관적인 **Tools**만 사용합니다.

---

## 이 예제에서의 MCP

이 프로젝트의 MCP 서버(`mcp_server/server.py`)는
메모에 대한 **도구 5개**를 AI에게 제공합니다.

| 도구 | 하는 일 | CRUD |
|---|---|---|
| `create_note` | 메모 생성 | **C**reate |
| `list_notes` | 메모 전체 조회 | **R**ead |
| `get_note` | 메모 단건 조회 | **R**ead |
| `update_note` | 메모 수정 | **U**pdate |
| `delete_note` | 메모 삭제 | **D**elete |

사용자가 "회의 메모 만들어줘"라고 하면,
Gemini가 알아서 `create_note` 도구를 골라 호출합니다.
**어떤 도구를 언제 쓸지는 AI가 판단**하고, 우리는 도구만 잘 만들어 두면 됩니다.
