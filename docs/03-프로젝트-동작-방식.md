# 03. 이 프로젝트 동작 방식 (코드 따라 읽기)

> 대상: 실제 코드가 어떻게 맞물리는지 한 줄씩 이해하고 싶은 사람

앞 문서의 개념이 코드에서 **정확히 어디에** 있는지 짚어봅니다.

---

## 파일 한눈에 보기

| 파일 | MCP 역할 | 핵심 |
|---|---|---|
| `common/database.py` | (도구가 부르는) 실제 저장소 | SQLite CRUD 함수 |
| `mcp_server/server.py` | **MCP 서버** | `@mcp.tool()` 로 도구 5개 노출 |
| `app/gemini_mcp.py` | **MCP 클라이언트 + Gemini** | `tools=[session]` 자동 호출 |
| `app/main.py` | **Host (FastAPI)** | `/chat` (AI) + `/notes` (직접 REST) |

## 1단계 — 도구를 정의한다 (`mcp_server/server.py`)

```python
mcp = FastMCP("notes")

@mcp.tool()
def create_note(title: str, content: str = "") -> dict:
    """새 메모를 생성한다. ...(생략)..."""
    return database.create_note(title, content)
```

- `FastMCP("notes")` : "notes"라는 이름의 MCP 서버를 만든다.
- `@mcp.tool()` : 이 함수를 **AI가 호출 가능한 도구**로 등록한다.
- 이때 AI에게 전달되는 정보는 세 가지로 자동 추출된다.
  - **함수 이름** → 도구 이름 (`create_note`)
  - **타입 힌트** → 인자 스키마 (`title: str`, `content: str`)
  - **docstring** → 도구 설명 (AI가 "언제 이 도구를 쓸지" 판단하는 근거)

> 💡 그래서 docstring은 **사람이 아니라 AI를 위한 설명서**입니다. 명확할수록 AI가 잘 고릅니다.

마지막에 `mcp.run()` 이 stdio 서버를 띄웁니다.

## 2단계 — 서버에 연결한다 (`app/gemini_mcp.py`)

```python
server_params = StdioServerParameters(
    command=sys.executable,            # 현재 파이썬으로
    args=[_SERVER_PATH],               # server.py 를 실행
)

async with stdio_client(server_params) as (read, write):     # ① 프로세스 실행
    async with ClientSession(read, write) as session:        # ② 세션 생성
        await session.initialize()                            # ③ 핸드셰이크
        ...
```

- ① `stdio_client(...)` : `python mcp_server/server.py`를 **하위 프로세스로 실행**하고 입출력 파이프를 연다.
- ② `ClientSession` : 그 파이프 위에서 MCP 대화를 담당하는 **클라이언트**.
- ③ `initialize()` : 서버와 버전·기능을 맞추는 악수(handshake).

이 `session`이 바로 [02번 문서]의 **Client** 입니다.

## 3단계 — Gemini에 MCP를 통째로 넘긴다 (핵심 한 줄)

```python
response = await client.aio.models.generate_content(
    model="gemini-2.5-flash",
    contents=message,                       # 사용자 자연어
    config=types.GenerateContentConfig(
        system_instruction=SYSTEM_INSTRUCTION,
        tools=[session],                    # ★ MCP 세션을 도구로 직접 전달
    ),
)
```

`tools=[session]` 이 이 프로젝트의 **마법의 한 줄**입니다.
`google-genai` SDK가 이 세션을 받으면 [02번 문서]의 3~7단계를 **자동으로** 수행합니다.

1. 세션에 `tools/list` 를 보내 도구 목록을 가져온다.
2. 그 목록을 Gemini가 이해하는 형식으로 변환해 함께 보낸다.
3. Gemini가 "이 도구 써라"라고 하면, 세션으로 `tools/call` 실행.
4. 결과를 다시 Gemini에 전달하고 최종 답변을 받는다.

> 즉, 우리는 function-calling 루프를 직접 돌리지 않습니다.
> 도구만 잘 만들고 세션만 넘기면 끝.

## 4단계 — HTTP로 노출한다 (`app/main.py`)

### (A) AI 경로 — `/chat`

```python
@app.post("/chat")
async def chat(body: ChatRequest):
    answer = await chat_with_tools(body.message)   # 2~3단계 호출
    return {"reply": answer}
```

사용자의 자연어 → Gemini → MCP 도구 → 답변. **이 프로젝트의 목적지.**

### (B) 직접 경로 — `/notes`

```python
@app.post("/notes")
def create_note(body: NoteCreate):
    return database.create_note(body.title, body.content)
# GET / PUT / DELETE 도 동일하게 database.* 호출
```

AI 없이 사람이 직접 호출하는 평범한 CRUD입니다.
**MCP 도구와 똑같은 `database.*` 함수**를 부른다는 점에 주목하세요.

```
/chat   (자연어) ─► Gemini ─► MCP 도구 ─┐
                                        ├─► common/database.py ─► notes.db
/notes  (직접 호출) ───────────────────┘
```

같은 저장소를 공유하므로, AI가 만든 메모를 `GET /notes`로 즉시 확인할 수 있습니다.
이것이 MCP의 실질적 가치 — **사람과 AI가 같은 시스템을 함께 다룬다**는 것을 보여줍니다.
